# Rule: JT808

Derived from `docs/business/RAAD_Phase3.4_JT808_Technical_Design_v1.md`.

1. **This service never writes Business API tables directly.** It only publishes domain events
   (`DevicePositionReported`, `DeviceOnline`, `DeviceOffline`, `DeviceAlarmRaised`, command-result
   events) to the broker.
2. **Vendor dialect variation is isolated in an Anti-Corruption Layer.** A new vendor means a new
   adapter — never a change to the core parser/dispatcher/handlers.
3. **Backfilled data is flagged and time-honest.** Buffered positions (0x0704, late 0x0200) publish
   with their original timestamp and `backfill=true`; live views must only use `event_time ≈ now` so
   stale buffered data never corrupts the live map.
4. **Session state lives in Redis**, keyed by `device_id -> {node, vehicle_id, org_id, last_seen,
   auth_state}` — the shared source of truth enabling sharded, sticky device connections.
5. **Unknown/unauthenticated devices are rejected and audited**, never silently dropped without a
   trace.
6. **Command downlink is correlation-ID tracked** — every platform-issued command (0x9101, 0x9201/
   0x9205, 0x8300, 0x8103, 0x8105) must be traceable back to the requesting use-case and its result.
7. **Connectivity monitoring is heartbeat-driven.** Timeout emits `DeviceOffline`; reconnect emits
   `DeviceOnline` — these feed device-status monitoring and exception workflows directly.
