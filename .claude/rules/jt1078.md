# Rule: JT1078

Derived from `docs/business/RAAD_Phase3.5_JT1078_Technical_Design_v1.md`.

1. **Parents have zero reachable path to video, anywhere, ever.** This is a platform-wide invariant
   (D5), not a configurable setting. Authorization is enforced in the Business API before any
   signaling reaches this service.
2. **RAAD is not a video archive.** This service stores no video; the MDVR is the sole system of
   record. Only ephemeral session/port/token state (Redis) and control metadata (`video_sessions`,
   `playback_requests` in the Business DB) are persisted.
3. **Device signaling is not this service's job.** The Business API signals the physical device via
   JT808 command downlink (0x9101 live, 0x9201/0x9205/0x9202 playback, 0x9102 stop); this service
   only receives and repackages the resulting media stream.
4. **Concurrency is bounded.** Enforce a per-org and global max-concurrent-stream ceiling; requests
   beyond the ceiling queue or are refused with a clear message. Ports are pooled and reclaimed on
   teardown.
5. **Media is repackaged, never passed through raw.** JT1078 frames become WebRTC (primary,
   low-latency) or HLS/FLV (fallback); clients never speak JT1078 directly.
6. **Every session open/close is audited**, with actor, device, camera, and time.
